# Expand the Box v2 — operator and contributor orientation

This folder is **v2** of the digital Expand the Box course: the same course taught better. v1 proved the teaching content, the safety architecture, and the tool engineering. v2 adds what v1 lacked: a real front door and HTML shell, one evolving daily practice wired into every module, a retrieval layer inside the no-quiz stance, the canon and safety patches, a sanctioned solo path, and the operator/commerce layer that lets a real person launch a cohort.

The build plan is locked in [`_build/ARCHITECTURE.md`](_build/ARCHITECTURE.md) (§1's eight decisions are the law of this build). The audit it came from is [`_build/AUDIT.md`](_build/AUDIT.md).

## Doc hierarchy

**Doc 04 is the law, the Run-Book is the route.** `04 - Container and Gatekeeping Protocol.md` binds every cohort: screens, thresholds, gates, the cap. `05 - Operator Run-Book.md` sequences it for an operator; `Facilitator Resources/CM First-Cohort Run-Book.md` sequences it for the CM. Where any other file disagrees with Doc 04, Doc 04 wins. Where wording disagrees with the canonical-phrasing ledger in the Voice Guide, the ledger wins.

## Relationship to v1

v1 lives at [`../Digital Expand The Box May 2026/`](../Digital%20Expand%20The%20Box%20May%202026/) and is the **frozen template**: read-only, never modified, the reference for every rewrite. Copied folders keep v1 names so relative links survive. The four confirmed v1 canon bugs (high drama, bright/shadow causation, matrix definition, EHP position count) are corrected in v2 per the ledger; do not copy v1 wording for those distinctions.

## Contribution rules

1. **The [Voice and Style Guide](Facilitator%20Resources/Voice%20and%20Style%20Guide.md) is binding** for every file, human- or AI-written. Read it before drafting; run its pre-flight checklist before submitting.
2. **The canonical-phrasing ledger** (Voice Guide §6) controls every PM distinction. State distinctions in ledger-compatible words. The three flagged CORRECTIONS override all v1 text.
3. **No gamification, no quizzes.** No badges, streaks, scores, certificates, answer-checking, or right/wrong feedback, anywhere. The retrieval line in Doc 01 defines what recall practice is allowed (generation yes, grading no).
4. **Link convention:** relative paths, spaces percent-encoded (`Days/Day%2001%20-%20...`). Learner-facing files link only paths that ship in this folder. The link checker is a release gate.
5. **Consent is never persisted.** Every gated tool re-arms its consent block on every visit. No `etb_*consent*` key may ever be written to localStorage. This is a safety invariant, not a style choice.
6. Learner-facing markdown ends with the World Copyleft footer block, exactly as the Voice Guide §8 gives it.
